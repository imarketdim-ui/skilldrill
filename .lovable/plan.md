## Зависимости между этапами

```
Этап 1 (фундамент)
  ├── Booking statuses → Этап 2 (авто-подтверждение по shadow_score)
  ├── Деактивация услуг → Этап 3 (импорт каталогов)
  └── VIP/ЧС интеграция → Этап 2 (shadow_score + cross-logic)

Этап 2 (доверие и поиск)
  ├── Shadow score → Этап 3 (маркетинг-таргетинг)
  └── Cross-logic → Этап 3 (роли модератора для ручной проверки)

Этап 3 (расширение)
  └── Независим, строится поверх 1+2
```

## Принцип безопасности между этапами

- Новые таблицы добавляются, существующие не меняют структуру (только новые колонки с DEFAULT)
- Новые enum-значения через ALTER TYPE ... ADD VALUE (обратно совместимо)
- UI-компоненты добавляются как новые вкладки/секции, не заменяют существующие
- Каждый этап может быть развёрнут независимо без отката предыдущего
