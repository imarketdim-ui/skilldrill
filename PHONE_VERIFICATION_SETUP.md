# Phone verification setup

Чтобы SMS-подтверждение номера работало в опубликованной версии, в проекте нужно включить phone auth в Supabase через Lovable.

## Что должно быть настроено

1. В `Supabase Auth` должен быть включён вход/подтверждение по телефону.
2. Для проекта должен быть настроен SMS provider.
3. В разрешённых redirect URL должен оставаться текущий publish domain приложения.

## Что уже реализовано в коде

- В клиентских настройках есть отдельный SMS flow для телефона.
- Новый номер не сохраняется в `profiles.phone`, пока пользователь не введёт OTP-код.
- После успешного подтверждения:
  - обновляется `profiles.phone`
  - выставляется `profiles.phone_verified_at`
  - пересчитывается `user_scores.phone_verified`

## Что проверить после настройки в Lovable

1. Открыть `/settings` или клиентский кабинет.
2. Ввести новый номер телефона.
3. Нажать `Отправить код`.
4. Получить SMS.
5. Ввести код и нажать `Подтвердить`.
6. Убедиться, что в UI появился статус `Телефон подтверждён`.
7. Убедиться, что в карточке рейтинга фактор `Телефон подтверждён` отмечен как выполненный.

## Если не работает

Типовые причины:

- в Supabase не включён phone provider;
- в проекте не настроен SMS provider;
- опубликован старый build без актуального phone verification flow;
- SMS provider отклоняет отправку на тестовый номер;
- у пользователя осталась незавершённая старая попытка смены номера.
